Five steps. No shadows, no borders, no rounding. Surfaces separate by L-step only — every UI region in the system maps to exactly one rung. If a new surface needs inventing, the system has failed and the case should be escalated, not resolved locally.

Rules of elevation

  • No shadows. Depth is conveyed by L-step, never by blur or offset.
  • No borders for elevation. Borders are for rules and dividers only, not to suggest a raised surface.
  • Zero radius. Every rung ends in a 90° corner. Rounding is not a design tool in this system.
  • Skip one step only. RAISE on BASE is legal; PEAK on BASE is legal; PEAK on SINK is a smell.
  • No invented rungs. If your component needs a sixth step, the composition is wrong — refactor, don't add.
  • Inverts cleanly. Light-mode values invert the ladder automatically. Designers don't pick light/dark tones manually.
