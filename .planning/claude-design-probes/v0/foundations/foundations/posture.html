Posture is how the system behaves, not how it looks. Five axes — intensity, speed, tracking, density, reach — roll up into three blessed presets: QUIET, LIVE, CHARGED. Every downstream component reads the derived tokens; individual axes can be scrubbed on any subtree for ad-hoc dials. The frame never moves. The signal is what you tune.

Rules of posture

  • Posture is behavior, not appearance. It changes how the system acts — not what it is.
  • The frame never moves. No posture alters geometry, color semantics, type scale, or spacing stops.
  • Three presets, five axes. Prefer presets. Scrub individual axes only when composing for a narrow surface.
  • Derived tokens only. Components read --posture-motion-duration, not --posture-speed. Keeps math central.
  • Inherit on subtrees. Posture classes cascade. A CHARGED panel inside a LIVE page is allowed; the opposite feels wrong.
  • No new presets. QUIET, LIVE, CHARGED cover the space. Novel moods extend axes, not the preset roster.
