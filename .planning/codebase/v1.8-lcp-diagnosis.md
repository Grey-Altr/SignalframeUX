**Cold vs warm divergence (Pitfall A — Anton bimodal LCP):**

Within each viewport, the cold-state selector matches the warm-state selector exactly. **No bimodal LCP candidate divergence observed in this build.** Both viewports converge on the same LCP element regardless of whether Anton has loaded — i.e., the LCP candidate's identity is font-state-stable. This is consistent with `display: optional` semantics: the fallback metrics for the LCP candidate are not large enough to lose-and-recover the LCP crown across the swap-in. (Note: timing diverges — mobile cold = 180 ms / warm = 68 ms; desktop cold = 92 ms / warm = 116 ms — but the *element* is the same.)

**Across viewports, the LCP candidate diverges entirely:**

- **Mobile** (360×800): LCP = the THESIS `GhostLabel` (4% opacity Anton at 200–400 px, positioned absolute behind section content).
- **Desktop** (1440×900): LCP = the visible VL-05 magenta `//` slash span (the ScreenBlend overlay layered on top of the hero h1, opacity 0.25).

This cross-viewport divergence is the core diagnostic finding for Phase 60. The mobile LCP is a **decorative wayfinding glyph at low opacity** (the GhostLabel by design is "structural wayfinding, not decoration" — see `components/animation/ghost-label.tsx:6-9`). The desktop LCP is a **canonical brand moment** (the magenta `//` mid-mark of `SIGNALFRAME//UX`). Phase 60 LCP-02 candidate selection must therefore branch on viewport.

## 3. Diagnostic notes — no intervention ranking (D-02)

**Sync-script disambiguation (RESEARCH finding 6):**

The codebase has TWO distinct CLS-prevention mechanisms that look superficially similar:

1. **External render-blocker:** `public/sf-canvas-sync.js`
   - Loaded as `<script src="/sf-canvas-sync.js" />` from `components/layout/scale-canvas.tsx:143` (line numbers verified at Plan 03 capture time).
   - Single-line IIFE; queries `[data-sf-canvas]`, computes `inner.offsetHeight * (vw / 1280)`, sets parent height pre-paint.
   - Phase 59 CRT-01 targets this file for inline + delete (REQUIREMENTS.md AES line in v1.8 scope).
   - Currently render-blocking by design (commit f6225b4 contract).

2. **Inline IIFE:** `const scaleScript` in `app/layout.tsx:91` (definition) injected as an inline `<script>` tag in `<head>` via React's HTML-string injection prop at `app/layout.tsx:113` (line numbers verified at Plan 03 capture time; original plan cited `:90-117` — the actual range is `:91` definition + `:113` mount).
   - Multi-line IIFE; reads `window.innerWidth/innerHeight`, computes 7 `--sf-*` CSS custom properties (`--sf-content-scale`, `--sf-canvas-scale`, `--sf-nav-scale`, `--sf-nav-morph`, `--sf-hero-shift`, `--sf-frame-offset-x`, `--sf-frame-bottom-gap`), writes to `document.documentElement.style` before first paint.
   - Pre-existing inline script; computes initial canvas + nav scale + morph values before first paint.
   - NOT part of CRT-01 scope. Conflating these two would break CRT-01 measurement.

**These are two different things. Phase 59 CRT-01 plan-phase MUST disambiguate.** The external file scope is "delete the `.js` file + inline its IIFE into layout head"; the inline `scaleScript` is already inline and out of CRT-01 scope.

**Headless WebGL caveat (RESEARCH §Q2):**

LCP measurements in this diagnosis were captured under SwiftShader software WebGL (`playwright.config.ts:22-27`). Real-GPU LCP timing on a physical device may differ. Phase 62 VRF-04 (real-device verification) is the source of truth for production LCP; Plan 03 captures the synthetic-CI baseline.

**Reduced-motion path:**

Plan 03 spec does NOT enable reduced-motion (in contrast to Plan 02 baseline-capture spec). LCP measurement reflects the LIVE first-paint state, not the curated reduced-motion variant. This is intentional — Plan 02 owns the "what does it look like" capture; Plan 03 owns the "what's the LCP target identity" capture.

**No intervention ranking:**

Per CONTEXT D-02, this document does NOT rank the three Phase-60 LCP-02 candidate paths (a/b/c). Ranking lives in Phase 60 plan-phase RESEARCH.md, where intervention tradeoffs are properly contextualized against measurement, LHCI median (Phase 58 output), Phase 59 ship state, and real-device divergence.
