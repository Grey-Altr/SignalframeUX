01

INSTALL

ADD SIGNALFRAMEUX™ TO YOUR PROJECT. THE CORE PACKAGE INCLUDES ALL SIGNAL LAYER COMPONENTS, TOKENS, AND THE API SURFACE. FIELD LAYER IS INCLUDED BUT OPT-IN.

# NPM npm install @sfux/core @sfux/components @sfux/tokens # PNPM (RECOMMENDED) pnpm add @sfux/core @sfux/components @sfux/tokens # FIELD LAYER (OPTIONAL) pnpm add @sfux/field
REQUIRES REACT 18+ AND TYPESCRIPT 5+. TAILWINDCSS IS OPTIONAL BUT RECOMMENDED.
02

INITIALIZE

WRAP YOUR APP IN THE SFUX PROVIDER. PASS A CONFIG OBJECT TO SET YOUR THEME, TOKEN MODE, AND FIELD LAYER PREFERENCES.

import { SFUXProvider, createSignalframeUX } from '@sfux/core' const config = createSignalframeUX({ theme: 'dark', tokens: 'oklch', field: { enabled: true } }) export default function App() { return ( <SFUXProvider config={config}> <YourApp /> </SFUXProvider> ) }
03

USE_COMPONENTS

IMPORT ANY OF THE 340+ COMPONENTS. THEY AUTOMATICALLY INHERIT YOUR THEME AND TOKEN CONFIGURATION. SIGNAL COMPONENTS ARE DETERMINISTIC. FIELD VARIANTS ADD GENERATIVE EFFECTS ON TOP.

import { Button, Card, Input } from '@sfux/components' function Dashboard() { return ( <Card> <Input placeholder="SEARCH SIGNALS..." /> <Button variant="signal"> •TRANSMIT </Button> <Button variant="field" fieldEffect="shimmer"> ENGAGE FIELD </Button> </Card> ) }
04

ACTIVATE_FIELD

THE FIELD LAYER IS THE GENERATIVE EXPRESSION SYSTEM. ADD NOISE BACKGROUNDS, PARTICLE MESHES, GLITCH TEXT, AND REACTIVE CANVAS EFFECTS. THEY NEVER INTERFERE WITH THE SIGNAL LAYER'S READABILITY.

import { NoiseBG, GlitchText } from '@sfux/field' <NoiseBG intensity={0.3} reactive="cursor" color="oklch(0.6 0.28 330)" /> <GlitchText trigger="hover"> SIGNAL//FRAME™ </GlitchText>
FIELD EFFECTS AUTOMATICALLY RESPECT PREFERS-REDUCED-MOTION. USE fieldIntensity TO CONTROL VISUAL WEIGHT.
05

DEPLOY

SIGNALFRAMEUX™ IS FRAMEWORK-AGNOSTIC BUT OPTIMIZED FOR NEXT.JS + VERCEL. TREE-SHAKING ENSURES YOU ONLY SHIP THE COMPONENTS YOU USE. FIELD LAYER IS CODE-SPLIT AND LOADED ON DEMAND.

# BUILD FOR PRODUCTION pnpm build # BUNDLE ANALYSIS pnpm analyze # CORE: ~12KB GZIPPED # COMPONENTS: ~3KB PER COMPONENT (AVG) # FIELD: ~8KB BASE + ~2KB PER EFFECT
