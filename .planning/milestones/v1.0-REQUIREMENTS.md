# REQUIREMENTS — SignalframeUX v1.0 Craft & Feedback

## FRAME — Token Foundation

- [x] **FRM-01**: All spacing uses blessed stops only (4/8/12/16/24/32/48/64/96) — zero arbitrary values in codebase
- [x] **FRM-02**: Semantic typography aliases implemented (`text-heading-1` through `text-body`, `text-small`) with font/weight rules
- [x] **FRM-03**: Layout tokens defined and enforced (`--max-w-content`, `--max-w-wide`, `--max-w-full`, standard gutters)
- [x] **FRM-04**: Every CSS custom property has a declared fallback value to prevent silent visual failures
- [x] **FRM-05**: Color palette tiered and frozen — core 5 required, extended locked, expansion prohibited
- [x] **FRM-06**: `--vhs-*` tokens namespaced to `--sf-vhs-*` for consistent Signal token boundary
- [x] **FRM-07**: CVA variant prop standardized to `intent` across all SF components with `defaultVariants` on every CVA call
- [x] **FRM-08**: Print media styles — dark backgrounds invert, Signal layer suppressed, readable on paper

## FRAME — Primitives

- [x] **PRM-01**: SFContainer primitive enforces layout tokens (max-width, gutters, responsive behavior)
- [x] **PRM-02**: SFSection primitive with `data-section`, `data-section-label`, consistent vertical spacing
- [x] **PRM-03**: SFStack primitive enforcing blessed spacing stops for vertical rhythm
- [x] **PRM-04**: SFGrid primitive with column system and responsive breakpoints
- [x] **PRM-05**: SFText primitive enforcing semantic typography aliases
- [x] **PRM-06**: SFButton refinement — consistent with new `intent` standard, interaction feedback tokens wired

## SIGNAL — Expression

- [x] **SIG-01**: ScrambleText fires on every route entry for primary headings
- [x] **SIG-02**: Asymmetric hover timing across all interactive elements (100ms in / 400ms out)
- [x] **SIG-03**: Hard-cut section transitions replace soft fades
- [x] **SIG-04**: Staggered grid entry animation with 40ms stagger on scroll reveal
- [x] **SIG-05**: `[data-anim]` elements visible without JavaScript — CSS fallback ensures content never invisible if GSAP fails
- [x] **SIG-06**: Audio feedback palette — Web Audio API synthesized tones (gesture-gated, monophonic), mapped to interaction types (click confirm, toggle, error) *(Completed in v1.1 Phase 7, plan 07-01)*
- [x] **SIG-07**: Haptic feedback via Vibration API on supported devices — mapped to press/confirm/error patterns *(Completed in v1.1 Phase 7, plan 07-01)*
- [x] **SIG-08**: Idle state animation — subtle Signal layer activity when page is inactive (grain drift, slow color pulse) that rewards lingering *(Completed in v1.1 Phase 7, plan 07-02)*
- [x] **SIG-09**: Signature cursor detail — magenta crosshair with particle trail on canvas sections
- [x] **SIG-10**: Mobile Signal layer behavior specified — which generative elements collapse to static at mobile widths, what the static fallback looks like

## SIGNAL — Above-the-Fold

- [x] **ATF-01**: Hero at 1440x900 is a standalone SOTD jury moment — no scroll required to evaluate quality
- [x] **ATF-02**: Hero motion fires within 500ms of load
- [x] **ATF-03**: Component count claim resolved honestly (show what's built accurately, "growing" label if appropriate)
- [x] **ATF-04**: Error page (app/error.tsx) is a crafted FRAME+SIGNAL moment, not a functional fallback
- [x] **ATF-05**: Empty states for component browser, token explorer, and API explorer designed as first-class interaction moments
- [x] **ATF-06**: Reduced-motion experience QA'd as a standalone intentional design, not just "animations off"

## FLUENCY — DX Contract

- [x] **DX-01**: SF component scaffolding spec documented — file structure, CVA shape, barrel export pattern, required props, data attributes
- [x] **DX-02**: FRAME/SIGNAL import boundary explicit — `sf/` = FRAME, `animation/` = SIGNAL, data attributes bridge them
- [x] **DX-03**: Per-component JSDoc with usage example on all SF-wrapped components
- [x] **DX-04**: `registry.json` distribution surface for AI/CLI-driven component installation *(Completed in v1.2 Phase 11, plan 11-01)*
- [x] **DX-05**: API architecture foundation — `createSignalframeUX(config)`, `useSignalframe()` hook returning tokens/theme/motion controllers *(Completed in v1.2 Phase 13, plan 13-01)*

## FLUENCY — State & Persistence

- [x] **STP-01**: Session state persistence — component browser filters, token explorer tab, scroll position survive return visits within same session *(Completed in v1.2 Phase 14, plan 14-01)*
- [x] **STP-02**: Theme toggle during GSAP animation guarded — transition buffer prevents OKLCH/inline color conflicts
