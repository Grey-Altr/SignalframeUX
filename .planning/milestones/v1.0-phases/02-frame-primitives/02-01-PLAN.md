Create three new files following the canonical CVA + cn() + forwardRef pattern from sf-button.tsx.

**sf-container.tsx:**
- Import `cva`, `VariantProps`, `cn`, and `React`
- CVA base: `"w-full mx-auto px-[var(--gutter-sm)] md:px-[var(--gutter)]"`
- Variants: `width` with values `wide` → `"max-w-[var(--max-w-wide)]"`, `content` → `"max-w-[var(--max-w-content)]"`, `full` → `"max-w-[var(--max-w-full)]"`
- defaultVariants: `{ width: "wide" }`
- Interface: `SFContainerProps extends React.ComponentProps<"div">, VariantProps<typeof sfContainerVariants>`
- Use `React.forwardRef<HTMLDivElement, SFContainerProps>` with named function
- Render `<div>` element — NOT main (main can only appear once per page)
- Set displayName to `"SFContainer"`
- NO `'use client'` directive

**sf-section.tsx:**
- Import `cn` and `React` (no CVA needed — simpler prop-based approach)
- Props interface: `label?: string`, `bgShift?: boolean`, `spacing?: "8" | "12" | "16" | "24"` (blessed vertical stops only), plus `React.ComponentProps<"section">`
- Default spacing: `"16"` (py-16 = 64px)
- Use `React.forwardRef<HTMLElement, SFSectionProps>` with named function
- Render `<section>` with:
  - `data-section` always present (boolean attribute, no value)
  - `data-section-label={label}` — React omits when label is undefined
  - `data-bg-shift={bgShift ? "" : undefined}` — presence-only boolean
  - `className={cn(\`py-\${spacing}\`, className)}`
- Set displayName to `"SFSection"`
- NO `'use client'` directive

**sf-stack.tsx:**
- Import `cva`, `VariantProps`, `cn`, and `React`
- CVA base: `"flex"`
- Variants:
  - `direction`: `vertical` → `"flex-col"`, `horizontal` → `"flex-row flex-wrap"`
  - `gap`: `"1"` → `"gap-1"`, `"2"` → `"gap-2"`, `"3"` → `"gap-3"`, `"4"` → `"gap-4"`, `"6"` → `"gap-6"`, `"8"` → `"gap-8"`, `"12"` → `"gap-12"`, `"16"` → `"gap-16"`, `"24"` → `"gap-24"` (all 9 blessed stops mapped to Tailwind gap utilities)
- defaultVariants: `{ direction: "vertical", gap: "4" }`
- Interface: `SFStackProps extends React.ComponentProps<"div">, VariantProps<typeof sfStackVariants>`
- Use `React.forwardRef<HTMLDivElement, SFStackProps>` with named function
- Set displayName to `"SFStack"`
- NO `'use client'` directive

**align_items support for SFStack (Claude's discretion):**
- Add `align` variant: `start` → `"items-start"`, `center` → `"items-center"`, `end` → `"items-end"`, `stretch` → `"items-stretch"`
- defaultVariants includes `align: "stretch"`
