**sf-grid.tsx:**
- Import `cva`, `VariantProps`, `cn`, and `React`
- CVA base: `"grid"`
- Variants:
  - `cols`: `"1"` → `"grid-cols-1"`, `"2"` → `"grid-cols-1 md:grid-cols-2"`, `"3"` → `"grid-cols-1 md:grid-cols-2 lg:grid-cols-3"`, `"4"` → `"grid-cols-1 md:grid-cols-2 lg:grid-cols-4"`, `"auto"` → `"grid-cols-[repeat(auto-fill,minmax(280px,1fr))]"`

- Interface: `SFGridProps extends React.ComponentProps<"div">, VariantProps<typeof sfGridVariants>`

- Set displayName to `"SFGrid"`
- NO `'use client'` directive

**sf-text.tsx:**
- Import `cn` and `React` — NO CVA (variant map is simpler as a plain Record)
- Define types:
  - `TextVariant = "heading-1" | "heading-2" | "heading-3" | "body" | "small"` — this is the enforcement union
  - `TextElement = "h1" | "h2" | "h3" | "h4" | "h5" | "h6" | "p" | "span" | "label"`
- Define `variantClassMap: Record<TextVariant, string>`:
  - `"heading-1"` → `"text-heading-1"`, `"heading-2"` → `"text-heading-2"`, `"heading-3"` → `"text-heading-3"`, `"body"` → `"text-body"`, `"small"` → `"text-small"`
- Define `defaultElementMap: Record<TextVariant, TextElement>`:
  - `"heading-1"` → `"h1"`, `"heading-2"` → `"h2"`, `"heading-3"` → `"h3"`, `"body"` → `"p"`, `"small"` → `"span"`
- Interface: `SFTextProps extends React.HTMLAttributes<HTMLElement>` with:
  - `variant: TextVariant` — REQUIRED, no default (forces explicit choice)
  - `as?: TextElement` — optional override, defaults to semantic mapping
  - `className?: string`
- Use `React.forwardRef<HTMLElement, SFTextProps>` with named function
- Inside: `const Tag = as ?? defaultElementMap[variant]`
- Render `<Tag ref={ref as React.Ref<any>} className={cn(variantClassMap[variant], className)} {...props} />`
- Cast ref as `React.Ref<any>` to handle polymorphic element types (accepted pattern per RESEARCH.md Pitfall 1)
- Set displayName to `"SFText"`
- NO `'use client'` directive
- Export both the component AND the `TextVariant` type (useful for downstream consumers)
