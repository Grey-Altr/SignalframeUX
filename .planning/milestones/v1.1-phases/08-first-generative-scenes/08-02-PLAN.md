Create `components/animation/token-viz.tsx` as a `'use client'` component.

**Token data to visualize:**

Colors (5): `--color-background`, `--color-foreground`, `--color-primary`, `--color-secondary`, `--color-accent`

Spacing (9 blessed stops): `[4, 8, 12, 16, 24, 32, 48, 64, 96]` — these are pixel values from CLAUDE.md, not CSS vars. Render as proportional bars.

Typography (9 scale entries): Read `--text-2xs` through `--text-4xl` from globals.css. If these are not readable as CSS custom properties, use the known pixel values: `[9, 11, 13, 16, 19, 23, 28, 47, 80]` with labels `["2xs", "xs", "sm", "base", "lg", "xl", "2xl", "3xl", "4xl"]`.

**Component structure:**

```typescript
"use client";

import { useEffect, useRef, useCallback } from "react";
import { resolveColorToken } from "@/lib/color-resolve";
```

- Use `useRef<HTMLCanvasElement>(null)` for the canvas.
- Define constants for layout: `PADDING = 32`, `SWATCH_SIZE = 48`, `GAP = 16`, `SECTION_GAP = 48`.
- The canvas renders 3 horizontal sections stacked vertically:

**Section 1 — Colors:**
- Row of 5 color swatches (filled rectangles, zero border-radius per CLAUDE.md).
- Below each swatch: the CSS variable name in JetBrains Mono, `9px`, muted foreground color.
- Below that: the resolved hex value (convert RGB to hex string).

**Section 2 — Spacing:**
- 9 horizontal bars, each proportional to its pixel value. Max bar width = canvas width - 2 * PADDING. Scale factor: `barWidth = (value / 96) * maxBarWidth`.
- Bar color: foreground at 0.2 opacity. Bar height: 12px.
- Label to the left of each bar: `"4px"`, `"8px"`, etc. in JetBrains Mono, 9px.

**Section 3 — Typography:**
- 9 text samples, each rendered at its actual font size using Inter.
- Text content: the scale name + pixel size, e.g., `"2xs — 9px"`, rendered in Inter at that size.
- Color: foreground.

**Canvas sizing:**
- Width: match parent container width (read from `parentElement.clientWidth`).
- Height: compute dynamically based on content — sum of all section heights + gaps + padding.
- Handle DPR: `canvas.width = w * dpr`, `canvas.style.width = w + "px"`, `ctx.setTransform(dpr, 0, 0, dpr, 0, 0)`.

**Drawing function `draw()`:**
- Read all color tokens fresh via `resolveColorToken` on every draw call.
- Read foreground color for labels: `resolveColorToken("--color-foreground")`.
- Use `ctx.font = '9px "JetBrains Mono", monospace'` for labels.
- Use `ctx.font = '${size}px "Inter", sans-serif'` for typography samples.
- Zero border-radius on all rectangles (use `fillRect`, not `roundRect`).

**Lifecycle:**
- Wrap initial draw in `document.fonts.ready.then(() => draw())` to avoid system font flash.
- Add `MutationObserver` on `document.documentElement` with `{ attributes: true, attributeFilter: ["class", "style"] }`. On mutation, call `draw()`.
- Add `ResizeObserver` on the canvas parent to redraw on resize.
- Cleanup: disconnect MutationObserver, disconnect ResizeObserver.

**Reduced motion:** This component has no animation loop — it draws once and redraws on mutation/resize. No reduced-motion guard needed (it's already static).

**Render:**

```tsx
return (
  <div className="px-6 md:px-12 py-12">
    <h2 className="text-xs uppercase tracking-[0.2em] text-muted-foreground font-bold mb-6">
      TOKEN DIAGNOSTIC
    </h2>
    <canvas
      ref={canvasRef}
      role="img"
      aria-label="Design system token visualization showing colors, spacing, and typography scales"
      className="w-full"
    />
  </div>
);
```
