  <action>
Create new file `lib/signalframe-provider.tsx` with `'use client'` directive as the FIRST line.

**Imports:** React createContext, useContext, useState, useEffect from 'react'; gsap from 'gsap'; toggleTheme from '@/lib/theme'.

**Exported types:**
- `SignalframeUXConfig` — `{ defaultTheme?: 'light' | 'dark' | 'system'; motionPreference?: 'full' | 'reduced' | 'system' }`
- `UseSignalframeReturn` — `{ theme: 'light' | 'dark'; setTheme: (theme: 'light' | 'dark') => void; motion: SignalframeMotionController }`

**Internal type:**
- `SignalframeMotionController` — `{ pause: () => void; resume: () => void; prefersReduced: boolean }`

**Context:** `const SignalframeContext = createContext<UseSignalframeReturn | null>(null);`

**Factory function `createSignalframeUX(config: SignalframeUXConfig = {})`:**
Returns `{ SignalframeProvider, useSignalframe }`.

**SignalframeProvider component:**
- `useState` for `isDark`: SSR default `true`, client reads `document.documentElement.classList.contains('dark')` — NOT localStorage. This matches the inline blocking script in layout.tsx.
- `useState` for `prefersReduced`: SSR default `false`, client reads `window.matchMedia('(prefers-reduced-motion: reduce)').matches`.
- `useEffect` for motion preference: If `config.motionPreference === 'reduced'`, call `gsap.globalTimeline.timeScale(0)` and set `prefersReduced(true)`. If `'full'`, call `gsap.globalTimeline.timeScale(1)` and set `prefersReduced(false)`. If `'system'` (default), attach MQL listener for `prefers-reduced-motion` changes, set state on change, cleanup on unmount.
- `setTheme` function: reads current class from DOM, compares to requested theme, calls `toggleTheme(currentDark)` from `lib/theme.ts` if different, updates `isDark` state with return value.
- `motion` object: `pause: () => gsap.globalTimeline.pause()`, `resume: () => { if (!prefersReduced) gsap.globalTimeline.resume(); }` (guarded — no-op when reduced motion active), `prefersReduced` state value.
- Renders `<SignalframeContext.Provider value={value}>{children}</SignalframeContext.Provider>`.

**useSignalframe hook:**
- Reads `useContext(SignalframeContext)`.
- If `null`, throws `new Error('[SignalframeUX] useSignalframe() must be called inside a <SignalframeProvider>. Wrap your app root with createSignalframeUX(config) and mount the returned SignalframeProvider.')`.
- Returns the context value.

**Also export `useSignalframe` standalone** (reads from same `SignalframeContext`) so consumers can import directly without threading the factory return.

**JSDoc:** Add JSDoc to `SignalframeUXConfig`, `UseSignalframeReturn`, `createSignalframeUX`, and the `motion` object methods (especially `resume` documenting the reduced-motion guard).

**DO NOT:**
- Import any layout primitives (SFSection, SFContainer, etc.)
- Add token resolution (no OKLCH canvas probes)
- Replace or duplicate `lib/theme.ts` logic
- Accept non-serializable config values (no callbacks)

Keep the file under 100 lines. This is thin infrastructure.
