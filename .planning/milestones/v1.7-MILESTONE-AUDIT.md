## Requirement Resolutions (this audit only)

### Phase 45 — Token Bridge

#### TBR-03 — `--sfx-*` consumer override tier documented in MIGRATION.md → **Ratified** (technique-divergence)

**Spec text:** "`--sfx-*` consumer override tier documented in MIGRATION.md with full variable list"

**Shipped artifact:**

- `MIGRATION.md` ships three dedicated sections covering the contract:
  - `## TOKEN NAMESPACE MIGRATION (--color-* → --sfx-*)` — explains the rename rule and JS/TS migration path
  - `## CONSUMER CSS INTEGRATION` — `@layer signalframeux` cascade rule + override example
  - `## CD SITE INTEGRATION EXAMPLE` — concrete `cd-tokens.css` reference with token override pairs
- 18 `--sfx-` references in MIGRATION.md
- `45-02-SUMMARY.md` confirms: "MIGRATION.md has 66 --sfx- references and @layer signalframeux instructions ✓" (the higher count includes earlier draft, current shipped version slimmed to 18)

**Divergence note:** spec asked for a "full variable list" enumeration. MIGRATION.md instead documents the **rename rule** (every `--color-*` → `--sfx-*`, every legacy `--sf-*` design-token → `--sfx-*`) plus per-section examples. Functional coverage is equivalent: any consumer reading MIGRATION.md learns the namespace pattern and the override mechanism. Pattern matches HLF-01's technique-divergence ratification precedent on this branch.

#### TBR-04 — No SSR magenta flash → **Ratified** (architecture-shipping mode)

**Spec text:** "No SSR flash — dark mode `class=\"dark\"` server-rendered on CD `<html>`, no magenta primary visible during streaming"

**Shipped artifact:**

- `dist/signalframeux.css` wraps `:root` and `.dark` blocks in `@layer signalframeux { }` (per `scripts/wrap-tokens-layer.ts` build pipeline shipped in 45-01)
- `cd-tokens.css` ships unlayered `:root { --sfx-primary: oklch(0.96 0 0); ... }` overrides (achromatic white over magenta)
- `MIGRATION.md:122` documents the SSR-safety guarantee: "No SSR flash because the override CSS is loaded synchronously in the CSS import chain — the browser resolves the final `--sfx-*` values before first paint."

**Divergence note:** the spec phrases the requirement in terms of CD-site implementation details (`class="dark"` on `<html>`). The CD site is a separate downstream consumer of SignalframeUX. SF//UX itself ships the **architecture** that makes the no-flash contract holdable: unlayered consumer override CSS wins over layered SF tokens before first paint, regardless of whether the consumer toggles `class="dark"` server-side or client-side. Pattern matches PTL-01's dual-mode ratification precedent: the architecture ratifies in this repo; the consumer-side `class="dark"` SSR is a downstream contract on the CD repo, not directly observable here.
