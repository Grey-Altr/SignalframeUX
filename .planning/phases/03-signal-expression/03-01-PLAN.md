**Part A: [data-anim] catch-all CSS fallback (SIG-05)**

Add a catch-all rule AFTER the existing specific `[data-anim="..."]` initial state rules (after line ~1027 in globals.css, after the `[data-anim="cta-btn"]` block):

```css
/* ── Catch-all: any [data-anim] without a specific rule above stays visible.
   GSAP sets opacity:0 on init before animating — content never disappears if JS fails. ── */
[data-anim] {
  opacity: 1;
}
```

This rule has LOWER specificity than `[data-anim="section-reveal"]` (attribute value selector > attribute presence selector), so existing specific rules still apply. New `[data-anim]` values without explicit CSS get the visible fallback.

Also update the reduced-motion block (around line 987) to include the catch-all. Add `[data-anim]` to the selector list:

```css
[data-anim],
[data-anim="section-reveal"],
[data-anim="tag"],
[data-anim="comp-cell"],
[data-anim="cta-btn"],
.sf-hero-deferred {
  opacity: 1 !important;
  transform: none !important;
}
```
