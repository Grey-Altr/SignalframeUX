Create `components/animation/signal-motion.tsx` as a 'use client' component:

```typescript
"use client";
import { useRef } from "react";
import { useGSAP } from "@gsap/react";
import gsap from "gsap";
import { ScrollTrigger } from "gsap/ScrollTrigger";
import { cn } from "@/lib/utils";

gsap.registerPlugin(ScrollTrigger);

interface SignalMotionProps {
  children: React.ReactNode;
  className?: string;
  from?: gsap.TweenVars;
  to?: gsap.TweenVars;
  scrub?: boolean | number;
  start?: string;
  end?: string;
  markers?: boolean;
}
```

**Implementation:**
- Default `from`: `{ opacity: 0.3, y: 20 }`
- Default `to`: `{ opacity: 1, y: 0 }`
- Default `scrub`: `1` (smooth 1-second catchup — this is the key differentiator from ScrollReveal which is `once: true` entrance)
- Default `start`: `"top 80%"`
- Default `end`: `"top 30%"`
- Use `useGSAP` with `{ scope: containerRef }`:
  - `gsap.fromTo(innerRef.current, from, { ...to, scrollTrigger: { trigger: containerRef.current, start, end, scrub, markers } })`
- Reduced-motion check: if `matchMedia('(prefers-reduced-motion: reduce)').matches`, skip animation entirely (children render at `to` state immediately)
- Render: `<div ref={containerRef} className={cn(className)}><div ref={innerRef}>{children}</div></div>`

This is distinct from ScrollReveal because:
1. ScrollReveal fires once on entry — SignalMotion scrubs with scroll position
2. ScrollReveal uses gsap.from — SignalMotion uses gsap.fromTo with scrub
3. Animation reverses when scrolling back up (scrub behavior)
