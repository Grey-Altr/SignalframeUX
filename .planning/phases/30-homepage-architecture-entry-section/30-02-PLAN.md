**Step 5: Add pointermove event listener** as a new `useEffect` hook. Place AFTER the `useGSAP` hook (around line 347):

```typescript
// EN-05: Pointer-driven FBM offset — subtle mouse interaction on ENTRY section
useEffect(() => {
  if (!hasWebGL) return;
  const container = containerRef.current;
  if (!container) return;

  // No mouse interaction in reduced-motion mode
  if (window.matchMedia("(prefers-reduced-motion: reduce)").matches) return;

  const handlePointerMove = (e: PointerEvent) => {
    if (!uniformsRef.current) return;
    const rect = container.getBoundingClientRect();
    const x = (e.clientX - rect.left) / rect.width;
    const y = 1.0 - (e.clientY - rect.top) / rect.height;
    uniformsRef.current.uMouse.value.set(x, y);
  };

  container.addEventListener("pointermove", handlePointerMove, { passive: true });

  return () => {
    container.removeEventListener("pointermove", handlePointerMove);
  };
}, [hasWebGL]);
```

Key constraints:
- `pointermove` listener attached to `containerRef.current` (the GLSLHero div), NOT `window` or `document`. This scopes interaction to the ENTRY section only (Pitfall 6 from RESEARCH.md).
- Y coordinate flipped: `1.0 - (e.clientY - rect.top) / rect.height` — GLSL convention has Y=0 at bottom.
- `{ passive: true }` on the event listener for scroll performance.
- Reduced-motion: early return, no listener attached.
- Default `uMouse` value is `(0.5, 0.5)` — centered, producing zero FBM offset when no pointer interaction.
- No lerp/smoothing on mouse leave — when pointer leaves the container element, the last position is retained. The FBM drift via uTime naturally smooths any abrupt visual change.
