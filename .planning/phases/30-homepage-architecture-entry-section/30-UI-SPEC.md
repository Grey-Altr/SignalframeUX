## Interaction Contracts

### EN-04: Nav Visibility — Scroll-Triggered Hard Cut

**Initial state (0px scroll):**
```css
nav {
  visibility: hidden;
  opacity: 0;
  position: fixed;
  top: 0;
  z-index: var(--z-nav);
}
```

**After ENTRY leaves viewport (ENTRY bottom reaches viewport top):**
```css
nav {
  visibility: visible;
  opacity: 1;
}
```

**Transition:** None. Hard cut. Duration: `--duration-instant` (34ms) maximum — this is the DU channel-switch register. No slide, no fade, no translate.

**ScrollTrigger config:**
- `trigger`: `#entry` (the ENTRY section)
- `start`: `"bottom top"` (ENTRY's bottom edge reaches viewport top)
- `toggleActions`: `"play none none reverse"`
- `onEnter`: set `visibility: visible; opacity: 1`
- `onLeaveBack`: set `visibility: hidden; opacity: 0`

**Accessibility:** Nav remains in DOM with `visibility: hidden` (not `display: none`) — skip-nav link and keyboard navigation remain intact in the accessibility tree.

**CSS classes to add to `globals.css`:**
```css
.sf-nav-hidden {
  visibility: hidden;
  opacity: 0;
}
.sf-nav-visible {
  visibility: visible;
  opacity: 1;
}
```
Replace `sf-nav-roll-up` with these two classes.

### EN-05: Shader Mouse Interaction

**Behavior:** Mouse position over ENTRY section modulates FBM offset or grid density in `GLSLHero`. Effect is subtle — the shader breathes in response to cursor, it does not warp dramatically.

**New uniform:** `uMouse` (`THREE.Vector2`, normalized 0–1) added to `GLSLHero` fragment shader.

**Interaction spec:**
- Pointer moves over ENTRY section → update `uMouse` via `mousemove` event listener on the section container
- Normalize: `x = event.clientX / window.innerWidth`, `y = 1.0 - event.clientY / window.innerHeight` (flip Y for GLSL convention)
- Apply in GSAP ticker: modulate `uTime` drift direction or FBM offset by `uMouse * 0.3` (30% max influence)
- Mouse leaves section: smoothly lerp `uMouse` back to `vec2(0.5, 0.5)` center over 2 seconds
- Reduced-motion: mousemove listener NOT attached — static fallback already handles this

**Visual result:** The noise field subtly leans toward the cursor position. Grid density is unaffected by mouse — only FBM offset shifts. This preserves the geometric structure while letting the noise field respond.

## States and Fallbacks

### ENTRY Section States

| State | Behavior |
|-------|----------|
| Default (WebGL available, motion allowed) | GLSLHero renders FBM + grid + Bayer dither; mouse interaction active |
| WebGL unavailable | Static div with `--color-primary` at 10% opacity (`HeroStaticFallback`) |
| `prefers-reduced-motion: reduce` | `HeroStaticFallback` — no WebGL loop, no mouse listener |
| 0px scroll | Nav is `visibility: hidden` |
| ENTRY section fully scrolled past | Nav is `visibility: visible` |

### Hero Title States

| State | Value |
|-------|-------|
| Initial render (SSR + hydration) | `opacity: 0.01` — visible enough for LCP measurement, invisible to eye |
| Reveal (post-hydration) | `clip-path` or opacity transition to `opacity: 1` over `--duration-glacial` (600ms) |
| Reduced motion | Rendered at `opacity: 1` immediately — no reveal animation |

## Accessibility Constraints

- Nav `visibility: hidden` (not `display: none`) — keyboard focus and skip-nav links remain operable at 0px scroll.
- Hero title `<h1>` — must be the only `<h1>` on the page. Stub section labels use `<h2>`.
- `GLSLHero` container: `aria-hidden="true"` (existing, correct — decorative shader).
- Hero title: explicit `aria-label` not needed — plain text `SIGNALFRAME//UX` is readable by screen readers.
- Mouse interaction (EN-05): pointer-only, no keyboard equivalent needed — purely decorative modulation.
- Focus management: no modals or panels in Phase 30. No focus trap changes needed.
