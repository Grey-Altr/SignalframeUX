  <action>
Per CONTEXT.md: minimal 'use cache' adoption as proof-of-concept on /system page. Discretion on granularity (page-level vs component-level).

**Decision: Page-level 'use cache' on /system page** — simplest PoC, lowest risk, proves the directive works. The page is already a static Server Component with no dynamic data, so caching is a no-op functionally but validates the Next.js 16 cache component pipeline.

1. Enable cache components in next.config.ts:
   - Add `cacheComponents: true` to the `experimental` block:
   ```typescript
   const nextConfig: NextConfig = {
     experimental: {
       optimizePackageImports: ["lucide-react"],
       cacheComponents: true,
     },
     // ... redirects unchanged
   };
   ```

2. Add `'use cache'` directive to app/system/page.tsx:
   - Add `'use cache';` as the FIRST line of the `TokensPage` function body (after the function signature, before the return):
   ```typescript
   export default async function TokensPage() {
     'use cache';
     return (
       // ... existing JSX unchanged
     );
   }
   ```
   - Note: `'use cache'` requires the function to be `async` — add `async` keyword to the function signature
   - The `metadata` export stays outside the function and is unaffected

3. Run `pnpm build` — must exit 0
   - If build fails with cache-related errors, check:
     a. Is `cacheComponents: true` in the correct location (under `experimental`)?
     b. Does the function need to be async?
     c. Are there client component imports that conflict with caching? (TokenTabs is a client island — should be fine as it's a child, not the cached component itself)

4. Run `pnpm start` and verify `/system` page loads correctly:
   ```bash
   curl -s -o /dev/null -w "%{http_code}" http://localhost:3000/system
   ```
   - Must return 200

5. Run the full Playwright suite one final time to confirm no regressions from cache component:
   ```bash
   pnpm exec playwright test
   ```

If 'use cache' causes ANY test regression or build failure, REVERT the change entirely (remove cacheComponents from config, remove 'use cache' from page). The zero-regression gate (MG-02, MG-03) takes absolute priority over the cache PoC.
  </action>
