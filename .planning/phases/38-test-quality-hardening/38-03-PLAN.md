    - .planning/phases/38-test-quality-hardening/38-RESEARCH.md (Pattern 5: husky + lint-staged setup, anti-pattern about tsc on individual files)
  </read_first>
  <action>
    1. Initialize husky: `pnpm exec husky init`
       This creates `.husky/pre-commit` with a default content.

    2. Replace `.husky/pre-commit` content with:
       ```sh
       pnpm lint-staged
       ```
       (Just the one command — lint-staged orchestrates both eslint and tsc)

    3. Add lint-staged config to package.json:
       ```json
       "lint-staged": {
         "*.{ts,tsx}": [
           "eslint --max-warnings=0"
         ]
       }
       ```
       NOTE: The RESEARCH recommends `bash -c 'tsc --noEmit'` in lint-staged, but `tsc --noEmit` is project-level (not file-scoped per RESEARCH anti-pattern warning). Run it as a separate step in .husky/pre-commit instead for clarity.

    4. Update `.husky/pre-commit` to:
       ```sh
       pnpm lint-staged
       pnpm tsc --noEmit
       ```
       This way: lint-staged runs ESLint on staged *.{ts,tsx} files only (fast), then tsc runs the full project type check (correct behavior per RESEARCH — TypeScript checking is cross-file).

    5. Verify the hook is executable: `ls -la .husky/pre-commit`

    6. Add `.husky/_` to .gitignore if husky init created a `_` directory (husky 9 typically doesn't).
  </action>
