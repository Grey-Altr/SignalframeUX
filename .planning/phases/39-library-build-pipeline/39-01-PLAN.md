**1. Refactor lib/signalframe-provider.tsx (per D-01 + D-07 conflict resolution — Option 2 from RESEARCH.md):**

Replace the static `import gsap from 'gsap'` (line 4) with a dynamic import pattern. The provider uses three GSAP APIs:
- `gsap.globalTimeline.timeScale(n)` — in the motionPreference useEffect
- `gsap.globalTimeline.pause()` — in the motion controller
- `gsap.globalTimeline.resume()` — in the motion controller

Refactor approach:
```typescript
// REMOVE: import gsap from 'gsap';
// ADD: lazy GSAP loader that no-ops when gsap is not installed

async function getGsap() {
  try {
    const mod = await import('gsap');
    return mod.default ?? mod;
  } catch {
    return null;
  }
}
```

In the useEffect that reads motionPreference, replace synchronous `gsap.globalTimeline.timeScale(...)` calls with:
```typescript
getGsap().then(gsap => {
  if (!gsap) return;
  gsap.globalTimeline.timeScale(matches ? 0 : 1);
});
```

In the motion controller object, replace:
```typescript
const motion: SignalframeMotionController = {
  pause: () => { getGsap().then(g => g?.globalTimeline.pause()); },
  resume: () => {
    if (!prefersReduced) getGsap().then(g => g?.globalTimeline.resume());
  },
  prefersReduced,
};
```

Keep the `import { toggleTheme } from '@/lib/theme'` — theme.ts has no gsap dependency. Keep all exported types unchanged (SignalframeUXConfig, UseSignalframeReturn, createSignalframeUX, useSignalframe).
