    ```markdown
    # MIGRATION GUIDE

    Migrating from internal SFUX app usage to the published `signalframeux` npm package.

    ## IMPORT PATH CHANGES

    | Old (internal) | New (npm package) |
    |----------------|-------------------|
    | `@/components/sf/sf-button` | `signalframeux` |
    | `@/components/sf/sf-accordion` | `signalframeux/animation` |
    | `@/lib/signal-canvas` | `signalframeux/webgl` |
    | `@/hooks/use-signal-scene` | `signalframeux/webgl` |
    | `@/hooks/use-scramble-text` | `signalframeux` |
    | `@/lib/signalframe-provider` | `signalframeux` |
    | `@/lib/utils` (cn) | `signalframeux` |
    | `@/lib/theme` (toggleTheme) | `signalframeux` |
    | `@/app/globals.css` | `signalframeux/signalframeux.css` |

    ## PEER DEPENDENCIES

    ```sh
    # Core (required)
    npm install signalframeux react react-dom

    # Animation components (optional)
    npm install gsap @gsap/react

    # WebGL components (optional)
    npm install three
    ```

    ## TOKEN CSS SETUP

    Import the token CSS in your app entry:
    ```tsx
    import "signalframeux/signalframeux.css"
    ```

    This provides all CSS custom properties: --sf-* colors, spacing stops, typography scale, animation tokens. Requires Tailwind CSS v4 in your project.

    ## COMPONENT API CHANGES

    None — all SF component props, variants (CVA `intent`), and behavior are identical. The only change is the import path.

    ## NOTES

    - `@sfux/*` namespace imports (seen in older documentation) never existed as published packages. Use `signalframeux` directly.
    - Type exports (TextVariant, SignalframeUXConfig, etc.) are available from the same entry points as their parent modules.
    ```
