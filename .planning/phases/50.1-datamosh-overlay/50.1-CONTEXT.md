<domain>
## Phase Boundary

Build an ambient datamosh overlay — site-wide pixel displacement + chromatic aberration that reads as "compression instability." Single-pass WebGL fragment shader registered with SignalCanvas singleton. Target: visible on close inspection, imperceptible at arm's length (same benchmark as grain).

</domain>

<decisions>
## Implementation Decisions

### Architecture (locked by xtop)
- **Do NOT build as a separate DOM overlay** like VHSOverlay
- Build `DatamoshOverlay` component that registers with existing SignalCanvas singleton via `useSignalScene`
- Effect runs as fullscreen fragment shader composited at canvas layer (z:-1) or fixed overlay canvas with `pointer-events: none`
- **No separate WebGL context** — use existing singleton

### Shader Approach (locked by xtop)
- Single-pass fragment shader combining:
  1. Micro-displacement — simplex noise-driven UV offset (uIntensity 0.002-0.004 range)
  2. Micro-channel separation — R/G/B sampled at slightly different offsets proportional to displacement
  3. Block quantization — optional `floor(uv * blockRes) / blockRes` at ~256 resolution for subtle codec macroblock feel
- uIntensity default: 0.003 (1-2px drift, "close inspection" zone)

### Integration (locked by xtop)
- Wire to `--signal-intensity` via MutationObserver cache pattern (same as SignalMesh INT-04)
- Map `uIntensity = signalIntensity * 0.006` so full intensity = 0.006 (still subtle)
- GSAP ticker drives `uTime` accumulation (same pattern as SignalMesh/GLSLSignal)
- GSAP breathing: slowly oscillate intensity between 0.001 and 0.004 over 8-12s cycles
- Respect `prefers-reduced-motion`: render single static frame or disable entirely
- Mount in `GlobalEffects` (`/components/layout/global-effects.tsx`)

### Performance Budget (locked by xtop)
- Single-pass fullscreen shader with 3 texture samples + noise = <0.5ms GPU
- Well within the <1ms budget

### What NOT to Do (locked by xtop)
- No SVG filters site-wide (not consistently GPU-composited, scroll jank)
- No VFX-JS (scroll lag, conflicts with GSAP scroll architecture)
- No optical flow (too expensive; noise displacement is visually indistinguishable at subtle intensities)
- No pre-baked datamosh video (for CD video deliverables, not ambient web overlay)
- No separate WebGL context (use existing SignalCanvas singleton)

### Claude's Discretion
- File location convention: `components/animation/datamosh-overlay.tsx` (follow existing convention) vs `packages/sfux/src/signal/DatamoshOverlay.tsx`
- Noise function choice (simplex, Perlin, or equivalent)
- Exact GSAP breathing curve parameters within the 8-12s range
- Block quantization threshold behavior at different intensity levels

</decisions>

## Specific Ideas

### Subtlety Spectrum (from xtop)
| uIntensity | Effect |
|------------|--------|
| 0.001 | Sub-pixel drift, visible only in screenshots |
| 0.003 | 1-2px drift, the target "close inspection" zone |
| 0.008 | Noticeable ambient motion |
| 0.02+ | Overt datamosh — too aggressive for ambient |

### GLSL Reference (from xtop)
Full fragment shader skeleton provided in ROUTING.md dispatch — use as starting point, adapt to project's Three.js/SignalCanvas patterns.

### Autoresearch Spec
- 3-5 iterations to tune: noise frequency, channel separation ratio, block grid, breathing curve
- Benchmark: at uIntensity=0.003, displacement visible at 100% zoom on close inspection but invisible in 5-second casual scroll
