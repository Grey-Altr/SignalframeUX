## Summary

DatamoshOverlay is a fullscreen fragment shader that registers with the SignalCanvas singleton using the same patterns already established in GLSLHero, GLSLSignal, and SignalMesh. The codebase has three confirmed implementations to copy from — there is no novel architecture to invent. The primary implementation task is writing a new GLSL fragment shader (displacement + channel separation + block quantization) and wiring it into the existing scaffolding.

The critical architectural fact is that DatamoshOverlay must NOT use a DOM position model like VHSOverlay. It must register with `useSignalScene` so the singleton's scissor/viewport render loop handles it. Because it is a fullscreen effect, its container div must be `fixed inset-0` (or equivalent fullscreen positioning), and the camera must be `OrthographicCamera(-1, 1, 1, -1, 0, 1)` with a `PlaneGeometry(2, 2)` quad — both confirmed patterns from GLSLHero and GLSLSignal.

The INT-04 MutationObserver cache pattern (module-level `_signalIntensity` variable updated by observer on `:root style` attribute changes) is the mandated approach for reading `--signal-intensity` without per-frame DOM calls. This pattern is implemented identically in both `glsl-hero.tsx` and `signal-mesh.tsx` and must be copied verbatim, not re-invented.

**Primary recommendation:** Copy the GLSLSignal structure (it is the simplest fullscreen shader in the codebase), replace its fragment shader with the datamosh shader from the xtop spec, add the INT-04 MutationObserver wiring from SignalMesh, add GSAP breathing tween, and add `<DatamoshOverlay />` to GlobalEffects alongside VHSOverlay.

function readSignalVars(): void {
  const style = getComputedStyle(document.documentElement);
  const raw = (name: string, fallback: number): number => {
    const v = parseFloat(style.getPropertyValue(name));
    return isNaN(v) ? fallback : v;

    attributeFilter: ["style"],

const _g = globalThis as unknown as { __sf_has_webgl?: boolean };

function checkWebGL(): boolean {
  if (typeof window === "undefined") return false;
  if (_g.__sf_has_webgl !== undefined) return _g.__sf_has_webgl;
  try {
    const canvas = document.createElement("canvas");

    _g.__sf_has_webgl = false;
    return false;
  }
}

## GLSL Shader Architecture

### Noise Function Available in Codebase
The project uses a **value noise** function in `glsl-hero.tsx` (hash-based smooth interpolation). There is NO simplex noise library imported — noise is inlined GLSL. For DatamoshOverlay, use the same inlined approach. Value noise is sufficient for UV displacement at this subtlety level (0.002-0.004 range).

Existing value noise from GLSLHero (confirmed in codebase):
```glsl
// Source: components/animation/glsl-hero.tsx
float hash(vec2 p) {
  return fract(sin(dot(p, vec2(127.1, 311.7))) * 43758.5453);
}

float noise(vec2 p) {
  vec2 i = floor(p);
  vec2 f = fract(p);
  vec2 u = f * f * (3.0 - 2.0 * f); // smoothstep
  return mix(
    mix(hash(i), hash(i + vec2(1,0)), u.x),
    mix(hash(i + vec2(0,1)), hash(i + vec2(1,1)), u.x),
    u.y
  );
}
```

For FBM, `glsl-hero.tsx` uses 4-octave accumulation. For datamosh displacement, 1-2 octaves is sufficient and cheaper.

### Fragment Shader Structure (from CONTEXT.md spec)
```glsl
precision highp float;

uniform float uTime;
uniform float uIntensity;   // 0.001–0.006 range (mapped from --signal-intensity * 0.006)
uniform float uBlockRes;    // block quantization resolution (default: 256.0)
uniform vec2  uResolution;

// [value noise functions: hash, noise — copy from glsl-hero.tsx]

void main() {
  vec2 uv = gl_FragCoord.xy / uResolution;

  // 1. Micro-displacement: noise-driven UV offset
  float n = noise(uv * 8.0 + vec2(uTime * 0.3, uTime * 0.17));
  vec2 offset = vec2(n, noise(uv * 8.0 + vec2(uTime * 0.17, uTime * 0.3))) * uIntensity;

  // 2. Block quantization (optional, subtle codec macroblock feel)
  vec2 blockUV = floor(uv * uBlockRes) / uBlockRes;
  vec2 sampleUV = mix(uv, blockUV, step(0.003, uIntensity) * 0.15);

  // 3. Channel separation — R/G/B at slightly different offsets
  vec2 uvR = sampleUV + offset;
  vec2 uvG = sampleUV + offset * 0.66;
  vec2 uvB = sampleUV - offset * 0.33;

  // 4. Sample — datamosh composites over whatever is drawn below
  // Since there is no scene texture (ambient overlay), output a very faint tint
  // at the displaced coordinates. Near-transparent: just chromatic fringe.
  float r = noise(uvR * 12.0 + uTime * 0.05) * uIntensity * 40.0;
  float g = noise(uvG * 12.0 + uTime * 0.05) * uIntensity * 40.0;
  float b = noise(uvB * 12.0 + uTime * 0.05) * uIntensity * 40.0;

  gl_FragColor = vec4(r, g, b, max(r, max(g, b)));
}
```

**Important:** The effect has no scene texture to sample — it is an ambient noise-chromatic effect. The output is a very low-opacity chromatic fringe driven by noise. This is the correct interpretation for an ambient overlay: it adds subtle colorimetric noise to the canvas layer, not a displacement warp of actual rendered content. At uIntensity=0.003, this will be barely perceptible — exactly the "close inspection zone" target.

## Common Pitfalls

### Pitfall 1: Building a DOM overlay instead of a scene
**What goes wrong:** Creating a `position: fixed` canvas element independently (like VHSOverlay's div approach) instead of using `useSignalScene`. Creates a second WebGL context on iOS Safari, causing context loss.
**How to avoid:** Use `useSignalScene` with a `fixed inset-0` container div — the SignalCanvas scissor system handles the rest.
**Warning sign:** If the component does NOT call `useSignalScene`, it is wrong.

### Pitfall 2: Per-frame getComputedStyle in ticker
**What goes wrong:** Reading `--signal-intensity` inside the GSAP ticker function via `getComputedStyle`. Causes hundreds of forced style recalculations per second.
**How to avoid:** Use the INT-04 module-level cache. The MutationObserver updates `_signalIntensity` when the `:root` style attribute changes; the ticker reads the cached value.
**Warning sign:** Any `getComputedStyle` or `style.getPropertyValue` inside a `gsap.ticker.add()` callback.

### Pitfall 3: Ticker accumulation without HMR guard
**What goes wrong:** In development, HMR causes the component to remount. Without `gsap.ticker.remove(tickerFn)` before `gsap.ticker.add(tickerFn)`, multiple ticker callbacks accumulate.
**How to avoid:** Always call `gsap.ticker.remove(tickerFn)` before `gsap.ticker.add(tickerFn)` in the same scope. Confirmed pattern in both GLSLSignal and SignalMesh.

### Pitfall 4: MutationObserver not disconnected on unmount
**What goes wrong:** TD-01 violation — detail views dramatically increase mount/unmount frequency. Observer accumulation causes WebGL jank.
**How to avoid:** Disconnect in the `useGSAP` cleanup return. Also null the module-level `_signalObserver` ref.

### Pitfall 5: uIntensity applied in wrong scale
**What goes wrong:** Passing `_signalIntensity` (0–1 range) directly as `uIntensity` instead of mapping: `uIntensity = _signalIntensity * 0.006`. At intensity=1.0, this produces 0.006 (still subtle), not 1.0 (full-screen warp).
**How to avoid:** Apply the mapping in the ticker: `uniformsRef.current.uIntensity.value = breathingBase * (1 + _signalIntensity * 2)` or similar. The breathing tween oscillates the base; intensity multiplies the ceiling.

### Pitfall 6: Wrong container positioning for fullscreen
**What goes wrong:** Using `absolute inset-0` (GLSLSignal/GLSLHero pattern for section-scoped scenes) instead of `fixed inset-0` for an ambient site-wide overlay. The scissor rect will only cover the current scroll position, not the full viewport.
**How to avoid:** Use `fixed inset-0` for the container div. The SignalCanvas is itself `fixed inset-0`.

### Pitfall 7: Breathing tween on uniform directly vs on intermediate object
**What goes wrong:** `gsap.to(uniformsRef.current.uIntensity, { value: 0.001, ... })` — if the uniform ref is null when GSAP tries to update it, this throws silently.
**How to avoid:** Tween an intermediate `breathingObj = { value: 0.003 }` object that is always alive; read from it in the ticker.

## Code Examples

### Component Skeleton
```typescript
// components/animation/datamosh-overlay.tsx
"use client";

import { useRef, useEffect, useState } from "react";
import * as THREE from "three";
import { useSignalScene } from "@/hooks/use-signal-scene";
import { gsap, useGSAP } from "@/lib/gsap-core";

// INT-04: module-level signal cache — same pattern as signal-mesh.tsx
let _signalIntensity = 0.5;
let _signalObserver: MutationObserver | null = null;

function readSignalVars(): void {
  const v = parseFloat(
    getComputedStyle(document.documentElement).getPropertyValue("--signal-intensity")
  );
  _signalIntensity = isNaN(v) ? 0.5 : v;
}

function ensureSignalObserver(): void {
  if (_signalObserver || typeof window === "undefined") return;
  readSignalVars();
  _signalObserver = new MutationObserver(readSignalVars);
  _signalObserver.observe(document.documentElement, { attributeFilter: ["style"] });
}

// WebGL check: globalThis cache (iOS Safari context limit)
const _g = globalThis as unknown as { __sf_has_webgl?: boolean };
function checkWebGL(): boolean { /* same as glsl-signal.tsx */ }

function DatamoshFallback() {
  return <div data-signal-scene="datamosh-static" className="fixed inset-0 pointer-events-none" aria-hidden="true" />;
}

function DatamoshWebGL() {
  const containerRef = useRef<HTMLDivElement>(null);
  const uniformsRef = useRef<{
    uTime: THREE.IUniform<number>;
    uIntensity: THREE.IUniform<number>;
    uBlockRes: THREE.IUniform<number>;
    uResolution: THREE.IUniform<THREE.Vector2>;
  } | null>(null);

  const buildScene = () => {
    const container = containerRef.current!;
    const camera = new THREE.OrthographicCamera(-1, 1, 1, -1, 0, 1);
    const scene = new THREE.Scene();
    const uniforms = {
      uTime:       { value: 0 },
      uIntensity:  { value: 0.003 },  // default: close inspection zone
      uBlockRes:   { value: 256.0 },
      uResolution: { value: new THREE.Vector2(container.clientWidth || window.innerWidth, container.clientHeight || window.innerHeight) },
    };
    uniformsRef.current = uniforms;
    const geo = new THREE.PlaneGeometry(2, 2);
    const material = new THREE.ShaderMaterial({
      vertexShader: VERTEX_SHADER,
      fragmentShader: FRAGMENT_SHADER,
      uniforms,
      transparent: true,
      depthWrite: false,
    });
    scene.add(new THREE.Mesh(geo, material));
    return { scene, camera };
  };

  useSignalScene(containerRef as React.RefObject<HTMLElement | null>, buildScene);

  useGSAP(() => {
    if (window.matchMedia("(prefers-reduced-motion: reduce)").matches) return;
    ensureSignalObserver();

    // Breathing: intermediate object so tween never touches a possibly-null ref
    const breathingObj = { value: 0.003 };
    const breathingTween = gsap.to(breathingObj, {
      value: 0.001,
      duration: gsap.utils.random(8, 12),
      ease: "sine.inOut",
      yoyo: true,
      repeat: -1,
    });

    // Ticker: uTime + intensity mapping (INT-04 — no DOM access here)
    const tickerFn = () => {
      if (!uniformsRef.current) return;
      uniformsRef.current.uTime.value += 0.016;
      // Map: uIntensity = breathingBase * (1 + signalIntensity * 2)
      // At default signal (0.5): range 0.001–0.004. At full signal (1.0): range 0.001–0.006.
      uniformsRef.current.uIntensity.value = breathingObj.value * (1 + _signalIntensity * 2);
    };
    gsap.ticker.remove(tickerFn);
    gsap.ticker.add(tickerFn);

    return () => {
      gsap.ticker.remove(tickerFn);
      breathingTween.kill();
      if (_signalObserver) { _signalObserver.disconnect(); _signalObserver = null; }
    };
  }, { scope: containerRef, dependencies: [] });

  return (
    <div
      ref={containerRef}
      data-signal-scene="datamosh-overlay"
      className="fixed inset-0 pointer-events-none"
      style={{ zIndex: -1 }}
      aria-hidden="true"
    />
  );
}

export function DatamoshOverlay() {
  const [hasWebGL] = useState(() => checkWebGL());
  const [isReducedMotion, setIsReducedMotion] = useState(() => {
    if (typeof window === "undefined") return false;
    return window.matchMedia("(prefers-reduced-motion: reduce)").matches;
  });
  useEffect(() => {
    const mq = window.matchMedia("(prefers-reduced-motion: reduce)");
    const handler = () => setIsReducedMotion(mq.matches);
    mq.addEventListener("change", handler);
    return () => mq.removeEventListener("change", handler);
  }, []);
  if (!hasWebGL || isReducedMotion) return <DatamoshFallback />;
  return <DatamoshWebGL />;
}
```

## Open Questions

1. **ResizeObserver for uResolution**
   - What we know: GLSLSignal uses ResizeObserver to update `uResolution` on container resize; GLSLHero also uses it.
   - What's unclear: For a `fixed inset-0` container, `container.clientWidth/Height` will always be the viewport size. ResizeObserver on a fixed element will still fire on viewport resize (window resize triggers reflow of fixed elements).
   - Recommendation: Use ResizeObserver on the container div — same as GLSLSignal. It handles the `window.resize` case via the normal DOM path.

2. **Interaction with VHSOverlay compositing**
   - What we know: VHSOverlay is CSS-only (fixed divs with backdrop/filter effects), z-index at `var(--z-vhs)`. DatamoshOverlay renders to the SignalCanvas at z:-1.
   - What's unclear: Whether the datamosh chromatic fringe reads visually against VHSOverlay's CRT scanlines.
   - Recommendation: Manual visual review is required at combined intensity 0.5 and 1.0. This is PRF-03 territory (visual coherence review), not a blocker for the phase.

3. **uResolution initialization for fixed fullscreen container**
   - What we know: `container.clientWidth` may be 0 on first render if the fixed div hasn't been measured yet.
   - Recommendation: Use `container.clientWidth || window.innerWidth` as fallback in `buildScene` — safe pattern.
