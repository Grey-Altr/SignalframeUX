## LCP Fast-Path Rationale

```
BEFORE (inside hydration tree):
<body>
  <WebVitals />
  <TooltipProvider>     ← client wrapper (Chrome waits for JS parse)
    <LenisProvider>     ← client wrapper (~700ms JS-parse on slow Android)
      <SignalframeProvider>
        <BorderlessProvider>
          <CdCornerPanel />  ← SSR'd but gated by hydration boundary
```

```
AFTER (direct <body> child):
<body>
  <WebVitals />
  <div class="sf-mesh-gradient" />
  <a href="#main-content">Skip to content</a>
  <CdCornerPanel />     ← SSR'd AND paints at HTML-parse time (no JS gate)
  <TooltipProvider>     ← hydration tree starts here (unchanged)
```

Chrome's LCP detector scores the wordmark `<text>` paint at HTML-parse time rather than waiting for the hydration boundary's blocking JS to resolve. The ~700ms JS parse on Moto G Power / 3G (synthesis §2 finding 2) is removed from the LCP critical path.
