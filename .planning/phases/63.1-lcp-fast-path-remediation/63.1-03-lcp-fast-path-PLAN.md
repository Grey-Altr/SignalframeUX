<objective>
Land the LCP fast-path that closes Phase 63's real-device FAIL verdict
(LCP 1711 ms / 3618 ms / 1851 ms across iPhone 14 Pro / Moto G Power 3G /
Moto G Stylus 4G — synthesis §1).

**Per RESEARCH §Plan 03 major finding:** `cd-corner-panel.tsx` is ALREADY a pure
Server Component (zero `'use client'`, zero React state — verified by reading
the 95-line file). The wordmark IS in the SSR HTML. The bottleneck is NOT
React-render-blocking; it's Chrome's **paint-after-blocking-JS-parse** sequencing
because `<CdCornerPanel />` is mounted INSIDE the hydration tree:

```
<body>
  <WebVitals />
  ...
  <TooltipProvider>     ← client wrapper
    <LenisProvider>      ← client wrapper, large
      ...
      <SignalframeProvider>
        <BorderlessProvider>
          <ScaleCanvas>{children}</ScaleCanvas>
          <Nav />
          <NavRevealMount />
          <CdCornerPanel />     ← SSR'd but inside hydration boundary
        </BorderlessProvider>
      </SignalframeProvider>
    </LenisProvider>
  </TooltipProvider>
```

Chrome's LCP detector waits for the body subtree's blocking JS to finish parsing
before it scores the paint of any element below the body's hydration boundary.
~700 ms of JS parse on slow Android (synthesis §2 finding 2) gates the wordmark
paint.

**The fix is to hoist `<CdCornerPanel />` to be a direct child of `<body>`,
BEFORE any client provider opens.** Pure HTML reorder — zero new code, zero
behavioral risk:
- Z-stacking: `z-[var(--sfx-z-nav,40)]` composes with `<Nav />` whether they're
  siblings under different providers or both under `<body>`. Stacking context
  preserved.
- `dark` class: `themeScript` writes to `document.documentElement` (i.e. `<html>`)
  at HTML-parse time; ancestor relationship preserved regardless of where
  `<CdCornerPanel />` mounts in `<body>`.
- `--sf-frame-bottom-gap` / `--sf-frame-offset-x`: `scaleScript` writes to
  `document.documentElement.style`; cascades to all descendants.
- `var(--sfx-cube-fill)`: CSS-var binding; preserved.
- Hydration mismatch risk: ZERO. CdCornerPanel is a pure Server Component with
  no JSX-stateful children.
- D-12 trademark fidelity: preserved verbatim. SVG `<mask>` knockout, kana
  see-through, English overlay, theme-hue cube fill — all intact.

Output: A clean atomic commit cohort on `chore/v1.7-ratification` ready to ship
as PR #3 of 3 (D-01 strict-bisect; D-02 strict-serial — Plan 02 tip is the base).
This is the phase's terminal commit cohort. Wave 0 also lands the WPT re-run
script + wordmark fidelity spec + Pitfall #10 ratio spec. Phase close-out includes
