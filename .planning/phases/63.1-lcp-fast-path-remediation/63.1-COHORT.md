## §7 Phase 64 hand-off scope (delta from original Phase 64 plan)

Phase 64 was previously scoped as "CRT-05 / 3-PR ship + D-10". Phase 63.1 close adds:

1. **Pitfall #10 synthetic baseline recalibration (D-09 successor)** — establish a synthetic baseline that's actually predictive of real-device measurements. Current Phase 60 LHCI 810ms is a localhost number; consider Vercel Speed Insights P75 or Catchpoint daily monitor as the new synthetic anchor.
2. **Next.js App Router framework chunk size investigation** — chunk 2979 = 168 KB / ~56 KB gz of framework runtime (ClientSegmentRoot, AppRouterAnnouncer, etc.). Consuming 1867ms TBT on Moto G Power 3G Fast = 53% of total TBT budget. Investigate `experimental.serverComponentsExternalPackages`, route segment splitting, or document as v1.x platform constraint.
3. **iPhone 14 Pro 4G LCP variance reduction** — n=3 median 2104ms with best run 1560ms shows high variance. Either move to a higher-n measurement source (Vercel Speed Insights field RUM) or extend Catchpoint quota for n=10+ medians.
4. **Anton font load discipline (ghost-label paint timing)** — runs 1+2 showed LCP=2104+ms with FCP at 1591/1912. The 211-513ms gap is the Anton swap moment for the ghost-label `<span class="sf-display">`. Same Anton-load-timing patterns from `feedback_anton_swap_size_band.md` apply here — preload + descriptor calibration may close this gap.

These four items absorb into Phase 64's existing 3-PR ship structure as separate plans, or escalate to v1.9 if Phase 64 stays narrow on its original CRT-05 scope.
