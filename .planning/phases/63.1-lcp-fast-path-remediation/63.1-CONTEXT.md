<decisions>
## Implementation Decisions

### Plan/PR shape

- **D-01:** **3 plans, 3 PRs** — one per fix vector (Plan 01 bundle reduction · Plan 02 JS deferral · Plan 03 LCP fast-path). Mirrors Phase 59 CRT-05 strict-bisect philosophy. Bundle / defer / fast-path are independent intervention surfaces; if any single PR regresses, clean bisect requires separation. Synthesis §4 lists 3 distinct vectors → 3 plans.
- **D-02:** **Wave order**: Plan 01 → Plan 02 → Plan 03 (strict serial). Each PR's LCP/TBT delta must be measured in isolation against the prior cohort tip; parallelism would conflate attribution.

### LCP fast-path strategy (Plan 03)

- **D-03:** **Two-pronged fast-path**: (b) **SVG-as-static-asset preloaded** + (d) **decouple wordmark from React hydration**.
  - Rejected (a) inline-critical-CSS — `experimental.inlineCss: true` breaks `@layer signalframeux` cascade ordering (Pitfall #7; vercel/next.js#47585).
  - Rejected (c) `font-display: optional` for JetBrains Mono on wordmark — current setting is `swap` with measured descriptors (no descriptor file yet; mirrors CRT-03 Anton precedent if needed). Tightening to `optional` would invisibly drop the wordmark glyph if font misses 100 ms window — degrades the aesthetic-of-record.
  - The bottleneck per synthesis §2 is **JS parser readiness gating SVG `<text>` paint**, not font load. Decoupling the wordmark from the React tree (render as static SSR'd inline SVG outside the hydration root, OR serve it as a preloaded static `.svg` asset) attacks the actual cost.

### Bundle reduction (Plan 01)

- **D-04:** **Aggressive route-level code-splitting + tree-shake animation libs** — GSAP / Lenis / ScrollTrigger lazy-loaded via dynamic imports on the homepage. Phase 61's 4 named chunk IDs (3302, e9a6067a, 74c6194b, 7525) MUST remain matched per `feedback_lockin_before_execute.md` and STATE.md "All 4 v1.7 named chunk IDs MATCHED" decision. Structural code-split preserves them while moving non-LCP-critical JS off the initial chunk.
- **D-05:** **No new runtime npm dependencies** — v1.8 standing constraint (REQUIREMENTS.md §Constraints; STATE.md). Bundle reduction works strictly via existing-package code-splitting and tree-shaking, not via dep swaps.

### JS deferral posture (Plan 02)

- **D-06:** **Dynamic imports for non-critical components** + **`requestIdleCallback` for animation hydration**. Anti-pattern guards from PITFALLS.md MUST hold:
  - **NEVER** `defer` or `async` `/sf-canvas-sync.js` (Pitfalls #2, #8) — it is an inline IIFE in `<body>` tail, must execute synchronously.
  - **NEVER** introduce a second rAF call site (Pitfall #5; single-ticker rule). Use the existing GSAP ticker or PerformanceObserver only.
  - **NEVER** flip `experimental.inlineCss: true` (Pitfall #7).
  - **PRESERVE** `@layer signalframeux` cascade ordering (Pitfall #7 corollary).

### Verification gate

- **D-07:** **Re-run vrf-01-* WPT/Catchpoint scripts on same 3 profiles, all 3 LCP <1000 ms.** Outputs land at `.planning/perf-baselines/v1.8/vrf-01-*-post-63.1.json`. This is the original VRF-01 success criterion the synthesis measured against; Phase 63.1 closes when those numbers cross under 1000 ms.
- **D-08:** **LHCI median-of-5 synthetic gate as intermediate per-PR check** — `.lighthouseci/lighthouserc.json` thresholds already enforce LCP ≤1000 ms (Phase 58 CIB-02). BND-04 stale-chunk guard (`rm -rf .next/cache .next` before measurement) MANDATORY before each WPT or LHCI run.
- **D-09:** **Pitfall #10 re-check on close-out** — confirm post-fix `real ÷ synthetic` ratios fall back under thresholds (LCP <1.3, TTI <1.5). If ratios still exceed, fix is incomplete even if absolute LCP <1000 ms.

### Aesthetic preservation gate

- **D-10:** **Visual baseline diff <0.5% per page** — Phase 60 LCP-03 precedent. Playwright cross-route diff against `.planning/visual-baselines/v1.8-start/` after each plan ships.
- **D-11:** **Cohort review via chrome-devtools MCP scroll-test** — Phase 60 D-08 precedent. Sign-off note dropped at `.planning/phases/63.1-lcp-fast-path-remediation/63.1-COHORT.md` after Plan 03 ships.
- **D-12:** **Wordmark glyph is a trademark primitive** — per `feedback_trademark_primitives.md` and AESTHETIC-OF-RECORD.md (cube-tile / pixel-sort / navbar-glyph trio). Static-SVG decoupling MUST preserve glyph rendering pixel-for-pixel; any visible font-metric drift (e.g. via fallback monospace stack rendering before JetBrains Mono swap) is a BLOCK.

### Claude's Discretion

- Exact SSR mechanism for wordmark decoupling (Plan 03 RESEARCH.md picks: Server Component carve-out · static `.svg` asset emission · or pre-hydration inline `<svg>` in `app/layout.tsx`).
- Tree-shake granularity for GSAP plugins — full ScrollTrigger lazy vs per-effect dynamic import (Plan 01 RESEARCH.md decides).
- `requestIdleCallback` polyfill posture for Safari (Plan 02 RESEARCH.md).
- Whether intermediate LHCI gate runs against a preview deployment URL or local prod-build; default = preview deployment (matches Phase 58 CIB-02 enforcement environment).

<specifics>
## Specific Ideas

- **Wordmark static SSR carve-out is preferred over `.svg` static asset** — keeping the SVG inline preserves currentColor / CSS custom property bindings (`var(--sfx-cube-fill)` at line 73) without re-plumbing through the asset pipeline. The static `.svg` asset path is the fallback if SSR carve-out introduces hydration mismatch warnings.
- **Decouple from React tree by hoisting the corner panel out of the hydration root entirely** — render as a Server Component or a static HTML fragment in `app/layout.tsx`'s `<body>`, BEFORE the React root mounts. Synthesis §2 ¶3 names this exact mechanism: "the wordmark `<text>` element should not block on the full JS payload."
- **Tree-shake target list candidates** — GSAP (`gsap`, `gsap/ScrollTrigger`, `gsap/SplitText` if used), Lenis (`@studio-freight/lenis` or `lenis`), Radix primitives not on the homepage, any `lucide-react` icons not in the masthead/hero. Plan 01 RESEARCH.md confirms the exact set against the bundle-analyzer treemap.
- **`requestIdleCallback` for animation hydration** — Plan 02 specifically; preserves single-ticker rule because the GSAP ticker keeps owning rAF, but its *initialization* is deferred to idle.
