Playwright class-chain selector escaping (CRITICAL):
- Tailwind arbitrary brackets and slashes appear LITERALLY in the rendered className DOMTokenList.
- Playwright's CSS engine requires escaping in `page.locator(...)` strings. Two safe patterns:
  1. Use `[class~="token"]` attribute substring matches per token (no escape required), composed:
     `span[class~="relative"][class~="top-[0.08em]"][class~="pr-[0.28em]"][class~="tracking-[-0.12em]"][class~="text-[1.28em]"]`
     — note `[class~=...]` matches whole-token by space-separated, so `top-[0.08em]` is a literal token match; brackets do not need CSS escaping inside attribute-value quotes.
  2. Use `page.locator('span').filter({ hasText: '//' })` for desktop and validate by reading `await loc.getAttribute('class')` then asserting toContain on substrings.
- Pattern (1) is deterministic across Tailwind regenerations; pattern (2) is robust to className reorder. Use BOTH in the spec — locate via (1), verify via class-attribute substring assertions.
