### Critical Finding: GSAP SSR Guard Divergence

Phase 41-02 added `typeof window !== "undefined"` SSR guards around all `gsap.registerPlugin()` calls in `lib/gsap-core.ts`, `lib/gsap-plugins.ts`, `lib/gsap-split.ts`, `lib/gsap-draw.ts`, `lib/gsap-flip.ts` — enabling consumer SSR safety (DIST-02).

Phase 43 **reverted** these guards from source, stating they are "unnecessary for the app." The current `dist/` was built before the revert and still has guards. But running `pnpm build:lib` (triggered by `prepublishOnly` during `npm publish`) will regenerate dist without guards, breaking `scripts/consumer-test.ts`.

**Fix:** Re-add SSR guards to the 5 gsap source files, or add a tsup banner/plugin that injects them during build.

**Affected requirements:** LIB-02, DIST-02
