## Integration Findings

### P0 — Critical: Homepage COMPONENTS ID Mismatch

**File:** `components/blocks/component-grid.tsx`
**Affected:** DV-03, SI-02

The homepage `COMPONENTS` array uses IDs `001`–`012` that map to a different component set than `COMPONENT_REGISTRY`. The registry was keyed against the ComponentsExplorer 34-item grid. When a homepage card is clicked, the wrong ComponentDetail data opens (e.g., clicking CARD opens TOGGLE's detail panel).

**Fix:** Remap homepage COMPONENTS `id` values to match their correct COMPONENT_REGISTRY keys, or look up by component name instead of numeric ID.

### P1 — Minor: SignalOverlay Z-Index Not Suppressed

**File:** `app/globals.css`
**Affected:** SI-04

The `[data-modal-open]` rule drops canvas cursor to z-10, but SignalOverlay toggle (z-210) remains above the detail panel. Not a hard break — overlay doesn't occlude panel content in normal use.

### P2 — Cosmetic: Waveform DocId Stale

**File:** `lib/component-registry.ts` entry `"102"`
**Affected:** DV-05, DV-06

Registry entry 102 `docId: "waveform"` resolves to pre-Phase-24 legacy doc with stale `importPath: "@sfux/signal"`. Should point to `"waveformSignal"`.
