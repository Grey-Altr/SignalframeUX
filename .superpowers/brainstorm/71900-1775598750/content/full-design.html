Core Design Principle

"Why doesn't every design system work this way?"

The site IS the design system performing. Every page is simultaneously: a visceral experience (gets you hired), a functional reference (developers use it), and a proof of craft (the components build the page you're reading).

API Reference — The Contract

/reference
No Prose. Tables, Code, Types.

Every SF component: name, typed props, defaults, variants, import path. Expandable to live preview + code. Instant search. Token API with copy-click on everything. Real usage patterns, not toy examples. A senior engineer scans this in 60 seconds and knows the system is real.

Get Started — Speed Is the Design

/start
"Install. Configure. Ship."

5 steps. Each: number + title + code block + one sentence. Code blocks are the star — large, highlighted, copy-button prominent. Zero to installed in under 3 minutes. No decoration between steps — just space and type.

Visual Language (All Pages)

Dark dominant. Near-black backgrounds. White/light foreground text. Magenta as the only accent.

Zero border-radius. Everything sharp. No rounded corners anywhere.

4px borders. Heavy structural lines between sections. The grid is visible.

SIGNAL layer present but controlled. Grain, OKLCH color shifts, generative textures — but they serve clarity, never fight it.

Typography-led. Anton for display. Inter for body. JetBrains Mono for code. Electrolize for system labels.

Density over whitespace. tDR poster density — every element earns its space.
